Flint AI Platform - API Integration Guide

Overview

Authentication

API Key Authentication

External partners authenticate using API keys issued per workspace. Include the key in all requests:

Authorization: Bearer <API_KEY>

Service Account

Each integration is associated with a service account that determines:

  • Accessible workspaces and resources
  • Permission levels (read, write, admin)
  • Rate limits and quotas

OAuth 2.0 (SSO Integration)

For user-context operations, Flint supports OAuth 2.0 with the following grant types:

  • Authorization Code (web apps)
  • PKCE (mobile/SPA apps)

Integration Options

Option 1: API Integration

Direct API calls for full control over the user experience. Best for:

  • Building custom chat interfaces
  • Embedding AI responses in your existing UI
  • Programmatic access to analytics and content

Option 2: iFrame Embed

Embed the Flint experience directly in your platform:

<iframe
  src="https://app.flintk12.com/embed/{workspace_id}?theme=light"
  width="100%"
  height="600px"
  allow="microphone; clipboard-write"
></iframe>

Customization parameters:

Parameter Description Values
theme UI color scheme light, dark, auto
locale Interface language en, ar, es, etc.
hide_header Hide navigation header true, false
activity_id Pre-load specific activity Activity UUID

Option 3: Whitelabel Solution

Full visual customization including:

  • Custom domain (e.g., tutor.adek.ae)
  • Custom logo and branding
  • AI assistant name and avatar
  • Custom color scheme

Core APIs

Base URL

https://api.flintk12.com/v1

Messages API

The Messages API is the core interface for AI tutoring interactions. Send student messages and receive AI tutor responses.

Send Message

POST /v1/chats/{chat_id}/messages 
POST /v1/chats/{chat_id}/messages
Content-Type: application/json
Authorization: Bearer <API_KEY>

{
  "content": {
    "type": "text",
    "text": "Can you explain the Pythagorean theorem?"
  },
  "user_id": "student_123",
  "metadata": {
    "grade_level": "8",
    "subject": "mathematics"
  }
}

Response (Streaming):

The response is delivered as Server-Sent Events (SSE) for real-time streaming:

event: message_start
data: {"message_id": "msg_abc123", "type": "assistant"}

event: content_delta
data: {"text": "The Pythagorean theorem states that"}

event: content_delta
data: {"text": " in a right triangle, a² + b² = c²"}

event: tool_use
data: {"tool": "whiteboard", "action": "draw_triangle"}

event: message_end
data: {"finish_reason": "complete", "usage": {"input_tokens": 150, "output_tokens": 320}}

Content Types

Messages can include various content types:

Type Description Example Use Case
text Plain text message Student questions, explanations
image Generated or uploaded images Visual aids, diagrams
code Code snippets with execution Programming lessons
whiteboard Interactive drawing canvas Math problems, diagrams
document PDF/Word documents Essay review, worksheets
presentation Slide decks Lesson summaries
video Generated educational videos Concept explanations
audio Voice messages Pronunciation, listening

Example: Multi-modal Message

{
  "content": [
    {
      "type": "text",
      "text": "Here's my attempt at solving this problem:"
    },
    {
      "type": "image",
      "source_url": "https://example.com/student-work.jpg"
    }
  ],
  "user_id": "student_123"
}

Interactive Tools

The AI tutor can invoke various tools during conversations to enhance learning:

Whiteboard

Interactive canvas for visual explanations and student work.

{
  "tool": "whiteboard",
  "parameters": {
    "action": "create",
    "template": "coordinate_plane"
  }
}

Code Execution

Run Python or JavaScript code with real-time output.

{
  "tool": "code",
  "parameters": {
    "language": "python",
    "code": "import matplotlib.pyplot as plt\nplt.plot([1, 2, 3], [1, 4, 9])\nplt.show()"
  }
}

Supported Languages:

  • Python (with NumPy, Pandas, Matplotlib)
  • JavaScript

Image Generation

Create educational visuals on-demand.

{
  "tool": "image",
  "parameters": {
    "prompt": "A labeled diagram of a plant cell showing organelles",
    "style": "educational_diagram"
  }
}

Visualizations

Generate charts and graphs from data.

{

  "parameters": {
    "type": "bar_chart",
    "data": {"labels": ["A", "B", "C"], "values": [10, 25, 15]},
    "title": "Student Survey Results"
  }

Generate slide decks for lesson summaries.

Webhooks & Events

Receive real-time notifications for important events.

Configure Webhook

POST /v1/webhooks 
{
  "url": "https://your-server.com/flint-webhook",
  "events": [
    "chat.created",
    "chat.completed",
    "message.sent",
    "moderation.alert"
  ],
  "secret": "your_webhook_secret"
}

Event Types

Event Description
chat.created New chat session started
chat.completed Chat session ended
message.sent Message sent (user or assistant)
moderation.alert Content flagged by moderation
activity.started Student began an activity
activity.completed Student completed an activity

Webhook Payload Example

{
  "event": "moderation.alert",
  "timestamp": "2026-04-29T14:30:00Z",
  "data": {
    "chat_id": "chat_def456",
    "message_id": "msg_xyz789",
    "user_id": "student_123",
    "category": "self_harm",
    "severity": "high",
    "action_taken": "response_blocked"
  }
}

Rate Limits & Best Practices

Rate Limits

Endpoint Limit
Messages API 100 requests/minute
Chat Operations 60 requests/minute
Activity Reads 120 requests/minute
Workspace Updates 10 requests/minute

Best Practices

  1. Use Streaming: For AI responses, always use streaming mode to provide real-time feedback to students.
  2. Cache Activity Data: Activities change infrequently; cache them locally to reduce API calls.
  3. Handle Reconnections: Implement exponential backoff for failed requests.
  4. Batch Operations: Use batch endpoints when fetching multiple resources.
  5. Webhook Verification: Always verify webhook signatures to ensure authenticity.

Error Handling

Error Response Format

{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The 'content' field is required",
    "details": {
      "field": "content",
      "reason": "missing_required_field"
    }
  }
}

Error Codes

Code HTTP Status Description
UNAUTHORIZED 401 Invalid or missing API key
FORBIDDEN 403 Insufficient permissions
NOT_FOUND 404 Resource not found
RATE_LIMITED 429 Too many requests
INVALID_REQUEST 400 Malformed request body
SERVER_ERROR 500 Internal server error
