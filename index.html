Flint API Spec

Overview

This document outlines an external API shape for education platforms that need to provision and manage Flint resources inside a larger product. The integration model is hierarchical: districts contain workspaces, workspaces contain terms, terms contain groups, and groups or terms contain activities and chats.

Flint is actively evolving, so the API surface described here is subject to refinement during implementation. This guide focuses on the customer-facing CRUD surface only: districts, workspaces, terms, groups, activities, and chats.

Authentication

The authentication model supports both service-to-service access and user-context access.

Service Account Authentication

Administrative provisioning and synchronization calls would typically use bearer token authentication.

Authorization: Bearer <API_KEY>

OAuth 2.0 / SSO

For user-context experiences, Flint can support OAuth 2.0 based SSO. The expected model is authorization code flow for server-rendered apps and PKCE for web or mobile clients.

This would allow a partner platform to launch Flint in the context of an authenticated end user without requiring separate sign-in.

Tokens would be scoped to the partner account and authorized resources. Write operations would typically be limited to administrative service accounts.

Conventions

Base URL

https://api.flintk12.com/v1

This base URL is illustrative and represents the intended external API namespace.

CRUD Semantics

Operation Pattern
List GET /v1/{resource}
Read GET /v1/{resource}/{id}
Create POST /v1/{resource}
Update PUT /v1/{resource}/{id}
Delete DELETE /v1/{resource}/{id}

In this lifecycle model, delete operations generally begin as soft deletion using deleted_at. Rows may then be permanently removed after the applicable data retention period.

Timestamps

All timestamps use ISO 8601 UTC strings.

Pagination

List endpoints are expected to support limit and offset.

Request and Response Shape

Write requests and single-resource responses use a resource-specific wrapper such as { "workspace": { ... } } or { "activity": { ... } }. List responses use { "data": [...], "pagination": { ... } }.

Common Fields

Field Meaning
id Stable unique identifier for the resource.
name Primary human-readable label for the resource where applicable.
creator_id Identifies the user or service account that originally created the resource.
visibility Controls whether a resource is hidden, unlisted, or broadly accessible within the tenant.
deleted_at Marks a resource for deletion before any later retention-based hard delete.

Resource Hierarchy

district
├── workspace
│   ├── term
│   │   ├── group
│   │   │   ├── group
│   │   │   ├── activities
│   │   │   │   └── chats
│   │   │   └── chats
│   │   ├── activities
│   │   │   └── chats
│   │   └── chats
│   └── activities   (optional library scope)
└── activities       (optional library scope)

Under the current Flint data model, a chat belongs to exactly one parent: term, group, or activity. An activity can belong to a group, a term, and in broader platform scenarios may also exist at workspace or district scope.

In this model, a workspace has an active_term_id. Workspace creation would also provision a default term and return both identifiers.

Districts

Districts are the top-level organizational container for a multi-workspace deployment.

Endpoints

GET/v1/districts

Lists districts available to the partner. Supports standard pagination such as limit and offset.

Example Response

{
  "data": [
    {
      "id": "district_public_school_network",
      "creator_id": "usr_admin_001",
      "name": "Public School Network",
      "tagline": "Unified AI learning platform",
      ...
    }
  ],
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 1
  }
}
GET/v1/districts/{district_id}

Returns one district object by ID.

Example Response

{
  "district": {
    "id": "district_public_school_network",
    "creator_id": "usr_admin_001",
    "name": "Public School Network",
    "tagline": "Unified AI learning platform",
    "organization_context": "Provide consistent, high-quality AI-supported learning across district-managed schools.",
    ...
  }
}
POST/v1/districts

Creates a district using a body shaped like { "district": { ... } }.

Example Request

{
  "district": {
    "id": "district_public_school_network",
    "creator_id": "usr_admin_001",
    "name": "Public School Network",
    "tagline": "Unified AI learning platform",
    "organization_context": "Provide consistent, high-quality AI-supported learning across district-managed schools.",
    ...
  }
}

Example Response

{
  "district": {
    "id": "district_public_school_network",
    "creator_id": "usr_admin_001",
    "name": "Public School Network",
    ...
  }
}
PUT/v1/districts/{district_id}

Updates district metadata using a body shaped like { "district": { ... } }.

Example Request

{
  "district": {
    "tagline": "AI learning platform for all schools",
    "organization_context": "Provide curriculum-aligned AI support across district-managed schools."
  }
}

Example Response

{
  "district": {
    "id": "district_public_school_network",
    "creator_id": "usr_admin_001",
    "tagline": "AI learning platform for all schools",
    ...
  }
}

Fields

Field Type Notes
id string Stable district identifier.
name string District display name.
tagline string Short descriptor shown in Flint surfaces.
logo_url string | null District logo.
custom_branding_enabled boolean Enables district-level branding behavior.
organization_context string High-level instructional or organizational context.
creator_id string Administrative owner at creation time.
deleted_at timestamp | null Soft deletion marker.

Workspaces

Workspaces are the primary tenant unit for managing schools, programs, or institution-level deployments.

Endpoints

GET/v1/workspaces

Lists workspaces. Typical filters include district_id together with pagination parameters.

Example Request

GET /v1/workspaces?district_id=district_public_school_network&limit=20&offset=0

Example Response

{
  "data": [
    {
      "id": "ws_central_secondary",
      "creator_id": "usr_admin_001",
      "district_id": "district_public_school_network",
      "name": "Central Secondary Schools",
      ...
    }
  ],
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 1
  }
}
GET/v1/workspaces/{workspace_id}

Returns one workspace object by ID.

Example Response

{
  "workspace": {
    "id": "ws_central_secondary",
    "creator_id": "usr_admin_001",
    "district_id": "district_public_school_network",
    "active_term_id": "term_default_2026",
    "name": "Central Secondary Schools",
    ...
  }
}
POST/v1/workspaces

Creates a workspace and provisions its default term using a body shaped like { "workspace": { ... } }.

Example Request

{
  "workspace": {
    "creator_id": "usr_admin_001",
    "name": "Central Secondary Schools",
    "district_id": "district_public_school_network",
    "visibility": "unlisted",
    "timezone": "UTC",
    ...
  }
}

Example Response

{
  "workspace": {
    "id": "ws_central_secondary",
    "creator_id": "usr_admin_001",
    "active_term_id": "term_default_2026",
    "name": "Central Secondary Schools",
    ...
  }
}
PUT/v1/workspaces/{workspace_id}

Updates workspace settings using a body shaped like { "workspace": { ... } }.

Example Request

{
  "workspace": {
    "timezone": "UTC",
    "organization_context": "Support multilingual, curriculum-aligned tutoring for grades 6-12.",
    ...
  }
}

Example Response

{
  "workspace": {
    "id": "ws_central_secondary",
    "creator_id": "usr_admin_001",
    "timezone": "UTC",
    ...
  }
}
DELETE/v1/workspaces/{workspace_id}

Deletes or deactivates a workspace. Final implementation may use soft-delete semantics.

Example Response

{
  "workspace": {
    "id": "ws_central_secondary",
    "creator_id": "usr_admin_001",
    "deleted_at": "2026-04-30T15:00:00Z",
    ...
  }
}

Fields

Field Type Notes
id string Stable workspace identifier.
district_id string | null Optional parent district.
active_term_id string Currently active term for the workspace.
name string Workspace display name.
color string Primary workspace brand color.
logo_url string | null Workspace logo.
visibility enum Workspace discovery/access posture.
timezone string | null Used for scheduling and access windows.
organization_context string Default instructional context.
grade_level_prompts array<object> Grade-specific prompt customizations.
moderation_guidelines string Tenant moderation policy text.
moderation_alert_types array<string> Moderation event categories to notify on.
allowed_ip_ranges array<string> Optional network allowlist.
access_schedule_enabled boolean Enables time-based access restrictions.
access_window_start_time string | null Access window start.
access_window_end_time string | null Access window end.
access_window_days array<string> Allowed days of week.
deleted_at timestamp | null Managed by dedicated delete flow.

Terms

Endpoints

Example Request

  "data": [
    {
      "id": "term_fall_2026",
      "creator_id": "usr_admin_001",
      "workspace_id": "ws_central_secondary",
      "name": "Fall 2026",
      ...
    }
  ],
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 1
  }
}
GET/v1/terms/{term_id}

Returns one term object by ID.

Example Response

{
  "term": {
    "id": "term_fall_2026",
    "creator_id": "usr_admin_001",
    "workspace_id": "ws_central_secondary",
    "name": "Fall 2026",
    ...
  }
}
POST/v1/terms

Creates a term using a body shaped like { "term": { ... } }.

Example Request

{
  "term": {
    "creator_id": "usr_admin_001",
    "workspace_id": "ws_central_secondary",
    "name": "Fall 2026",
    "start_date": "2026-08-24",
    "end_date": "2026-12-18",
    ...
  }
}

Example Response

{
  "term": {
    "id": "term_fall_2026",
    "creator_id": "usr_admin_001",
    "workspace_id": "ws_central_secondary",
    "name": "Fall 2026",
    ...
  }
}
PUT/v1/terms/{term_id}

Updates term metadata or lifecycle state using a body shaped like { "term": { ... } }.

Example Request

{
  "term": {
    "visible": true,
    "end_date": "2026-12-19",
    ...
  }
}

Example Response

{
  "term": {
    "id": "term_fall_2026",
    "creator_id": "usr_admin_001",
    "workspace_id": "ws_central_secondary",
    "visible": true,
    ...
  }
}
POST/v1/terms/{term_id}/activate

Marks a term as the active term for its workspace.

Example Response

{
  "term": {
    "id": "term_fall_2026",
    "workspace_id": "ws_central_secondary",
    "visible": true,
    ...
  },
  "workspace": {
    "id": "ws_central_secondary",
    "active_term_id": "term_fall_2026"
  }
}

Fields

Field Type Notes
id string Stable term identifier.
workspace_id string Owning workspace.
name string Term name such as "Fall 2026".
start_date date | null Academic start date.
end_date date | null Academic end date.
visible boolean Whether the term is visible to workspace users.
creator_id string Administrative owner.
deleted_at timestamp | null Soft deletion marker. Active terms cannot be deleted.

Chats

Chats store session-level tutoring interactions and can belong to an activity, a group, or a term.

Endpoints

GET/v1/chats

Lists chats. Typical filters include parent_activity_id, parent_group_id, or parent_term_id together with pagination parameters.

Example Request

GET /v1/chats?parent_activity_id=act_linear_equations_intro&limit=20&offset=0

Example Response

{
  "data": [
    {
      "id": "chat_linear_equations_intro",
      "creator_id": "usr_student_204",
      "parent_activity_id": "act_linear_equations_intro",
      "name": "Linear equations practice",
      ...
    }
  ],
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 1
  }
}

Returns one chat object by ID, including metadata and messages.

Example Response

{
  "chat": {
    "id": "chat_linear_equations_intro",
    "creator_id": "usr_student_204",

Error Handling

Error Format

{
  "error": {
    "code": "INVALID_REQUEST",
    "message": "The request body is missing a required field."
  }
}

Common Error Codes

Code HTTP Status Description
UNAUTHORIZED 401 Missing or invalid API credentials.
FORBIDDEN 403 Caller does not have permission to perform the operation.
NOT_FOUND 404 Referenced resource does not exist.
INVALID_REQUEST 400 Request body is malformed or missing required fields.
CONFLICT 409 Operation conflicts with current resource state.
SERVER_ERROR 500 Unexpected server error.
